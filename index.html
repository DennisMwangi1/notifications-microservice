The Challenge

Notification handling inevitably pollutes core business logic.

As platforms expand into multi-tenancy, developers face fragmented integrations across Email, SMS, and Push providers. Rate limits fail downstream requests, localized templates become unmanageable, and retry mechanisms clutter core microservices resulting in lost messages and poor observability.

The Nucleus Approach

A decoupled, event-driven engine acting as the single source of truth.

Nucleus acts as a standalone notification utility. Microservices simply fire and forget an event describing the intent. Nucleus intercepts this event in Go, buffers it instantly in Kafka, and delegates processing to Node.js workers that enforce tenant rules, compile MJML templates, and guarantee fault-tolerant delivery.

Core Capabilities

1

Isolated Multi-Tenancy

Every microservice or domain operates within strict boundaries. Nucleus maintains independent API keys, preferred SMS/Email provider integrations, and dedicated routing preferences for every registered tenant on the system.

2

Dynamic Provider Fallbacks

Stop worrying about SendGrid or Twilio outages. If an event attempts delivery via Email through Provider A and fails, the worker immediately escalates the payload to backup Provider B, enforcing guarantees for mission-critical alerts.

3

Resilient Failure Queues (DLQ)

Hard failures don’t vanish into the logs. Undeliverable payloads are routed to a Kafka Dead Letter Queue topic. The Nucleus Admin Dashboard provides full visibility enabling administrators to diagnose the error, modify the payload, and manually retry.

4

Omnichannel Templating

Nucleus utilizes robust MJML frameworks for generating heavily compatible responsive emails, bound seamlessly with Handlebars. Developers manage global template definitions centrally instead of burying raw HTML strings into service repositories.

5

Real-Time WebSockets

Support seamless, instant in-app alerts. Using Centrifugo channels secured via HMAC JWTs, users immediately see push updates in their browser or mobile app without expensive data-polling loops.

6

Bring Your Own Provider

A highly modular Adapter Registry in the Go Gateway allows clients to transparently plugin their own preferred CPaaS accounts (SendGrid, Twilio, Resend, Firebase) defined at the Tenant layer.

7

Zero-Trust Webhooks

Exposed ingestion paths mandate strictly isolated API keys per tenant alongside rigorous HMAC payload signature validations, stopping malicious or polluted alerts dead in their tracks before they ever hit the Kafka broker.

8

Cache-Backed Resiliency

Redis enforces stringent rate limits and payload idempotency constraints locally. This ensures backend services aren't DDOS'd by runaway core loops or retry errors from upstream microservices continuously dumping duplicate events.
